# Plan: Cyber Stray Web UI (赛博街溜子 Web 控制台)

## 目标
为 Cyber Stray Agent 构建一个符合 DESIGN.md 视觉规范的 Web 控制台，严格遵循 TypeScript + Next.js App Router + Tailwind CSS + Framer Motion 技术栈。

## 项目现状分析
- **后端**: Bun CLI Agent，状态持久化到 `data/state.json`，行动日志输出到 stdout
- **无 HTTP API**: 后端是纯命令行进程，无内置 Web 服务
- **设计规范**: DESIGN.md 已定义完整的 Cyber-Fluid 视觉系统（Catppuccin 色彩、字体、动效）
- **技术约束**: 无 `any` 类型、React Functional Components、Framer Motion spring 物理动画

---

## 执行步骤

### Step 1: 初始化 Next.js 子项目
在 `web/` 目录下创建独立的 Next.js 15 项目，避免与现有 Bun 后端冲突。

**命令:**
```bash
cd /home/zewang/PROJECTS/cyber-stray
npx create-next-app@latest web --typescript --tailwind --eslint --app --no-src-dir --import-alias "@/*"
```

**安装额外依赖:**
```bash
cd web
npm install framer-motion
npm install -D @types/node
```

**说明:**
- 使用 `--no-src-dir` 使文件结构扁平（`app/` 直接在根下）
- 根目录已存在 `package.json`，因此 Web UI 作为独立子项目，通过 `cd web && npm run dev` 启动

### Step 2: 设计系统配置
配置 Tailwind、全局字体和 Catppuccin 色彩系统。

**文件:**
- `web/tailwind.config.ts` — 扩展 Catppuccin Mocha/Latte 色板、自定义字体族、自定义动画关键帧
- `web/app/globals.css` — 全局暗色主题默认、字体平滑、滚动条样式
- `web/app/layout.tsx` — 通过 `next/font/google` 加载 Space Grotesk、Inter、JetBrains Mono

**设计令牌:**
| Token | Value |
|-------|-------|
| Spring | stiffness: 400, damping: 25 |
| 标题字重 | 700/800，letter-spacing: -0.04em |
| 正文行高 | 1.6 |
| 玻璃模糊 | backdrop-blur-md |
| 卡片悬浮 | translateY(-4px) + 霓虹边框发光 |

### Step 3: 全局布局与背景动效
构建应用的骨架和氛围层。

**组件:**
- `app/layout.tsx` — Root Layout，注入字体、主题 Provider
- `components/layout/Sidebar.tsx` — 左侧导航栏（玻璃态 Mantle 背景），含 Logo、导航链接
- `components/layout/ThemeToggle.tsx` — 暗/亮主题切换按钮
- `components/effects/CyberGridBackground.tsx` — 缓慢流动的网格线背景
- `components/effects/MouseGlow.tsx` — 跟随鼠标的柔和光晕（CSS radial-gradient 或 Canvas）

**类型:**
- `types/theme.ts` — `Theme: 'mocha' | 'latte'`
- `types/layout.ts` — `NavItem: { label: string; href: string; icon: React.ComponentType }`

### Step 4: 核心数据组件
与 Agent 状态相关的展示组件，全部使用显式接口。

**组件:**
- `components/dashboard/AgentStatusPanel.tsx` — 主仪表盘容器
- `components/dashboard/CircularGauge.tsx` — 环形进度条（boredom/energy/temper）
  - Props: `value: number; max?: number; color: string; label: string;`
  - 使用 SVG stroke-dasharray 实现，Framer Motion 动画数值变化
- `components/dashboard/MoodBadge.tsx` — 心情标签
  - Props: `mood: Mood;`
  - 不同 mood 映射不同颜色和微动画（如 excited 轻微弹跳）
- `components/dashboard/StatCard.tsx` — 统计卡片（总狩猎次数、总推送次数等）
  - 字体使用 JetBrains Mono，带连字

**类型复用:**
- 将根目录 `src/types.ts` 中的核心类型（`AgentState`, `Mood`, `ActionType`, `LogEntry` 等）复制或链接到 `web/lib/types.ts`
- **严禁**使用 `any`，所有组件 props 必须显式接口化

### Step 5: API 层（前后端桥梁）
由于后端无 HTTP 服务，通过 Next.js API Routes 直接读取文件系统。

**API Routes:**
- `app/api/state/route.ts` — `GET` 读取 `../../data/state.json`，返回 `AgentState`
- `app/api/history/route.ts` — `GET` 读取 `../../data/history/` 目录下的 JSON 文件列表
- `app/api/logs/route.ts` — `GET` 返回模拟/静态日志数据（未来可接入真实日志文件）

**Hooks:**
- `hooks/useAgentState.ts` — 封装 SWR/fetch，5 秒轮询自动刷新
- `hooks/useHistory.ts` — 获取历史推送数据

**说明:**
- API Routes 在服务端运行，可直接访问文件系统，无需 CORS
- 如果 `data/state.json` 不存在，返回 404 并附带友好错误信息

### Step 6: 页面路由与内容
使用 Next.js App Router 创建以下路由：

**`/` — Dashboard 首页**
- AgentStatusPanel（环形仪表盘 boredom + energy + temper）
- 最近心情与状态摘要
- 快速操作按钮（磁性按钮：手动触发 Hunt、投喂、安抚）
- 最近日志预览（TerminalLog 组件前 5 条）

**`/history` — 历史推送**
- FeedCard 瀑布流列表
- Staggered reveal 入场动画（Framer Motion `staggerChildren`）

**`/logs` — 终端日志**
- TerminalLog 全屏视图
- 打字机效果逐行输出，光标闪烁
- 支持按日志级别筛选（debug/info/warn/error）

**`/settings` — 配置展示（只读）**
- 展示当前 AgentConfig 关键配置项
- 说明文字提示修改需编辑后端 `.env`

### Step 7: 交互与动效组件
实现 DESIGN.md 中定义的高级交互。

**组件:**
- `components/ui/MagneticButton.tsx` — 磁性吸附按钮
  - Props: `children: React.ReactNode; strength?: number;`
  - 监听鼠标位置，用 Framer Motion `useMotionValue` + `useSpring` 实现偏移
- `components/ui/GlassCard.tsx` — 玻璃态卡片容器
  - Props: `children: React.ReactNode; glowColor?: string;`
  - 悬浮时边框渐变发光（Mauve → Blue），背景 `bg-opacity-10 backdrop-blur-md`
- `components/effects/RadarScan.tsx` — 雷达扫描加载动画（搜索状态使用）
- `components/effects/PulseBorder.tsx` — 红色呼吸灯脉冲（饥饿/警告状态使用）
- `components/ui/TypewriterText.tsx` — 打字机效果文字
  - Props: `text: string; speed?: number; onComplete?: () => void;`

### Step 8: 状态感知动效集成
将 Agent 状态与 UI 动效绑定。

**逻辑:**
- 当 `boredom >= 80` 时，页面边缘显示 `PulseBorder`（红色呼吸灯）
- 当 `mood === 'excited'` 时，MoodBadge 增加弹跳动画
- 当 `mood === 'grumpy' || mood === 'emo'` 时，界面整体色调略微偏移（降低饱和度）
- 数据刷新时，数值变化使用 `animate` 过渡，而非突变

### Step 9: 类型安全与代码质量
全局类型检查，确保零 `any`。

**检查项:**
- `web/tsconfig.json` 启用 `"strict": true` 和 `"noImplicitAny": true`
- 所有 API 响应定义显式接口（如 `ApiResponse<T>`）
- 所有组件 props 定义 `interface`
- 所有 hooks 返回类型显式标注

### Step 10: 构建与验证
验证 Web UI 可独立运行并与后端数据正常对接。

**验证步骤:**
1. `cd web && npm run build` — 确保无 TypeScript 错误、无构建失败
2. `cd web && npm run lint` — ESLint 通过
3. `cd web && npm run dev` — 启动后访问 `http://localhost:3000`
4. 确认 Dashboard 正确显示 `data/state.json` 中的 Agent 状态
5. 确认所有动画运行流畅，无性能卡顿
